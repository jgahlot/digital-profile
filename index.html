Quoting Policy Chatbot

  • Copilot Studio
  • Microsoft Teams
  • RPP data

What I built — A Copilot Studio AI agent that provides instant, policy-based answers to support Renewal and Quoting teams during live customer conversations.

Who it helps

Renewal Specialists, Quoting Specialists, and stakeholders who need fast access to policy and configuration details.

When to use

When a specialist needs quick policy clarification, RPP data lookup, or fast validation support without manual file searching.

How it works
  1. Ask a question in the chatbot (Teams / Copilot).
  2. Retrieve policy answers and RPP details (including item-number configurations).
  3. Upload old vs. new quotes to compare differences (quote validation support).
Inputs

User questions, RPP data sources / files, and optional quote files (old and new).

Outputs

Instant answers, configuration details, and quote comparison results.

Tools / Integrations

Copilot Studio; Microsoft Teams / Copilot. Extensible to Salesforce, Microsoft tools, Oracle, and other systems.

Status / Notes: Designed for fast access during calls and to reduce manual searching in large files.

IB Review AI Agent

  • Abacus AI
  • Copilot Studio
  • SPARK
  • Weekly refresh

What I built — An AI agent that monitors at-risk customers using the IB Review data file in SPARK and surfaces actionable insights for leaders.

Who it helps

Geo leaders, managers, and stakeholders who need visibility into customer risk signals captured by CSMs.

When to use

When reviewing customer health / risk trends and prioritizing outreach across regions and districts.

How it works
  1. Consumes the IB Review file from SPARK (CSM call notes / inputs).
  2. Classifies accounts — Concerning / Cautious as at-risk; Neutral / Positive otherwise.
  3. Publishes refreshed insights after the weekly data update.
Inputs

IB Review data file in SPARK; CSM-provided customer notes / signals.

Outputs

Customer risk classification and an at-a-glance view for monitoring by Geo / district.

Tools / Platforms

Implemented in both Abacus AI and Copilot Studio to compare platform fit and user experience.

Refresh cadence: Source file is refreshed every Monday; I redeploy the agent weekly for updated insights.

IB Review AI Workflow

  • Abacus AI
  • Copilot CLI
  • Excel
  • Automated email

What I built — An automated workflow version of the IB Review AI Agent that proactively emails at-risk customer details to Renewals Geo leaders.

Who it helps

Renewals Geo leaders and managers who need scheduled visibility into at-risk accounts.

When to use

When you want proactive, recurring distribution of at-risk accounts (instead of users opening a tool and checking manually).

How it works
  1. Identifies at-risk accounts (Concerning and Cautious).
  2. Filters by Renewals Geo (APAC, EMEA, AMER).
  3. Generates three separate “At-Risk” Excel files — one per Geo.
  4. Drafts a tailored email per Geo and sends it with the file attached.
Inputs

IB Review dataset; classification logic; Geo mapping for APAC / EMEA / AMER.

Outputs

3 Geo-specific Excel files + 3 emails sent to the appropriate Geo leaders.

Platforms

Abacus AI (scheduled trigger, daily emails) and Copilot CLI (local command to trigger and send emails).

Status / Notes: Copilot CLI is a good short-term option (runs locally); Abacus AI supports fully scheduled delivery.

PVR Justification AI Agent

  • Abacus AI
  • Copilot Studio
  • PDF parsing

What I built — An AI agent that helps the Renewals team validate PVR requests before submission by checking whether the request aligns with defined policy criteria.

Who it helps

Renewals team members preparing PVR submissions and seeking faster, policy-aligned validation.

When to use

Before submitting a PVR, to quickly assess whether the justification and details are aligned with approval criteria.

How it works
  1. User uploads the PVR document(s).
  2. The agent reviews the PVR PDF and checks it against the decision conditions.
  3. The agent returns a policy-based assessment of whether the request is likely to meet the criteria.
Inputs

PVR documents (PDF) and defined policy criteria / decision conditions.

Outputs

A structured assessment indicating alignment with criteria, supporting decision-making before submission.

Tools / Platforms

Built in both Abacus AI and Copilot Studio.

Notes: Uses decision conditions I trained based on information provided by Herb.

$0 PVR Tracking & Review AI Agent

  • Copilot Studio
  • Microsoft Teams
  • SharePoint
  • Power Automate

What I built — A Copilot Studio AI agent in Microsoft Teams that helps the Renewals team track $0 PVR approvals end-to-end through a conversational experience.

Who it helps

Renewals team, Geo leaders, and stakeholders monitoring approvals, rejections, pending items, and aging.

Problem / Current state

Approvers respond via email; responses are captured in a SharePoint list via Power Automate. Stakeholders then review the list manually to track status.

How it works
  1. User asks the chatbot for PVR status and approval progress.
  2. The agent retrieves relevant records from the SharePoint list (captured via Power Automate).
  3. The agent summarizes status in a clear, conversational format.
Inputs

SharePoint list entries (approval status) populated by Power Automate from email responses.

Outputs

Real-time tracking answers in Teams — approvals, pending, rejections, dates, and overall status.

Tools / Integrations

Copilot Studio + Microsoft Teams; SharePoint List + Power Automate for status capture.

Status / Next steps: Prototype ready and working; awaiting Renewals team feedback before rolling out to the full team.

Quote Validation Tool

  • Python
  • Oracle (thick client)
  • Browser-local
  • Modernization

What I built — Modernized a legacy Python-based quote validation solution and rebuilt it to run in a browser locally, with a plan to deploy it on a server for broader use.

Who it helps

Teams that need a scalable quote validation capability without per-user installation overhead.

Problem / Current state

The IBX team shared a legacy codebase with outdated libraries. It previously ran as local scripts requiring installation on each machine, limiting usage to individual setups.

What I changed
  1. Reviewed the full codebase and upgraded it to the latest Python version.
  2. Removed outdated libraries and updated the Oracle database connection.
  3. Rebuilt the solution to run locally in a browser.
Key challenge

Many legacy libraries were only compatible with earlier Python releases; upgrading and validating the end-to-end solution took ~1 month.

Dependencies

Oracle database client installation was required during testing. The legacy solution used a thin client; the updated program runs on the Oracle thick client.

Status / Next steps

Runs successfully on my local machine. Next: deploy to a virtual server so others can use and test it.

Notes: Identifying the right team at NetApp to support server deployment.
